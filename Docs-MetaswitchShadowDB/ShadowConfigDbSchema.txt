ShadowConfigDbSchema.txt
------------------------

This file documents the structure of the Shadow Configuration Database, and the
mapping of Sh field names to database column names.

For details concerning the use of the information in this file please consult
the 'MetaView Shadow Configuration Database' chapter of the MetaSphere EAS,
CommPortal and MetaView Reporting Manual.


==================
Database Structure
==================
The 'public' schema contains Shadow Configuration Database for the local
MetaView Server (the server that you've connected to).

There is also a 'cluster' schema, which allows access to the same data as the
'public' schema, but aggregated across a cluster of MetaView Servers.  This can
be used to make cluster wide queries, e.g. for a clustered CFS deployment.


==============
Access details
==============

Database software: PostgreSQL 13.10
Port: 5432
Name: shadow_config_db
Username: shadowconfigread
Obtain the password from your secure administrator, who can configure the password in the 'Configure database passwords' craft option.

Access is restricted to read-only views, hereafter referred to as the 'tables'.

===============
Table structure
===============

A table in general represents a Service Indication (SI) or group of SIs that
can logically be stored together. For example, the Meta_Subscriber table
contains a number of subscriber types (individual line, business group line
PBX etc.)

Each table can have the following field types:
                row_id   a primary key for the record
       <Sh field name>   a field name generated from the SI name and Sh
                         field name as specified in the map below
 <Sh field name>usedef   a boolean field indicating whether the
                         corresponding field is inheriting its value from
                         elsewhere e.g. a subscriber inheriting from its
                         persistent profile. The absence of this field
                         indicates that the field cannot inherit. A null
                         value indicates that the field is not valid
                         e.g. a call service is not configured.
   <Non-Sh field name>   a field representing an identifier or index, or
                         otherwise a field that is not available over Sh.


For more documentation on Service Indications and the Sh fields within them
please see the MetaView SOAP/XML interface schema, serviceData.xsd.

The following Service Indications are not currently exposed over the
published SOAP/XML interface. For documentation on these please see the
Operations Manual for the appropriate object.
(Note that the Sh field names listed in this file for these SIs are linked
to the field names displayed in MetaView Explorer.)
(Also note that not all fields visible in MetaView Explorer have columns
in the database.)
